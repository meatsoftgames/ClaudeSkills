# Create Unity Project

When the user asks to create a new project, follow these steps exactly.

## Step 1: Gather Info

Ask the user two questions:
1. **Project name** — what should the project/repo be called?
2. **Project type** — is this a "package" (Unity package for reuse) or a "consumer" (standalone Unity project)?

## Step 2: Create GitHub Repo

Create a private repo under the user's GitHub account:

```bash
"/c/Program Files/GitHub CLI/gh.exe" repo create mattsmithone/<project-name> --private --description "<project-name> - Unity project" --confirm
```

## Step 3: Clone to Correct Location

- **Consumer projects** clone to: `E:\Personal\<project-name>`
- **Package projects** clone to: `E:\Personal\Packages\<project-name>`

```bash
"/c/Program Files/GitHub CLI/gh.exe" repo clone mattsmithone/<project-name> "<target-path>"
```

## Step 4: Copy Template Project

Copy the contents of the Unity template project into the cloned repo:

```bash
cp -r "E:/Personal/Project Init/TemplateProject/"* "<target-path>/"
```

Do NOT copy `.git/`, `.sln`, or `.csproj` files from the template — those are generated by Unity.

```bash
cd "<target-path>" && rm -f *.sln *.csproj
```

Download the standard Unity `.gitignore` from GitHub (the template may not have one):

```bash
curl -sL "https://raw.githubusercontent.com/github/gitignore/main/Unity.gitignore" -o "<target-path>/.gitignore"
```

## Step 5: Update Project Settings

Edit `<target-path>/ProjectSettings/ProjectSettings.asset`:
- Set `productName` to the project name
- Set `companyName` to `MeatSoft`

Use `sed` to replace these values:
```bash
sed -i "s/productName: .*/productName: <project-name>/" "<target-path>/ProjectSettings/ProjectSettings.asset"
sed -i "s/companyName: .*/companyName: MeatSoft/" "<target-path>/ProjectSettings/ProjectSettings.asset"
```

Also ensure `runInBackground: 1` is set:
```bash
sed -i "s/runInBackground: 0/runInBackground: 1/" "<target-path>/ProjectSettings/ProjectSettings.asset"
```

## Step 6: Package Scaffolding (package projects only)

If the project type is "package", create the standard Unity package structure:

```
<target-path>/Packages/com.meatsoft.<package-name>/
├── package.json
├── Runtime/
│   ├── <PackageName>.Runtime.asmdef
│   └── (empty, ready for code)
├── Editor/
│   ├── <PackageName>.Editor.asmdef
│   └── (empty, ready for code)
└── Tests/
    └── Editor/
        ├── <PackageName>.Tests.asmdef
        └── (empty, ready for tests)
```

The `package.json` should contain:
```json
{
  "name": "com.meatsoft.<package-name>",
  "version": "0.1.0",
  "displayName": "<Project Name>",
  "description": "",
  "unity": "6000.0",
  "author": {
    "name": "MeatSoft"
  }
}
```

The Runtime asmdef:
```json
{
  "name": "<PackageName>.Runtime",
  "rootNamespace": "MeatSoft.<PackageName>",
  "references": ["Unity.Mathematics", "Unity.Collections", "Unity.Burst"],
  "includePlatforms": [],
  "excludePlatforms": [],
  "allowUnsafeCode": false,
  "overrideReferences": false,
  "precompiledReferences": [],
  "autoReferenced": true,
  "defineConstraints": [],
  "versionDefines": [],
  "noEngineReferences": false
}
```

The Editor asmdef should reference the Runtime asmdef. The Tests asmdef should reference both Runtime and Editor, plus NUnit.

## Step 7: Create Obsidian Project

Create a project folder and index page in the user's Obsidian vault using the `mcp__obsidian__write_note` tool.

### Folder name
Use the project name converted to kebab-case with capital first letters (e.g. "VoxelTerrain" → "Voxel-Terrain", "TerrainGen" → "Terrain-Gen").

### Create `<Project-Name>/_index.md`

Use `mcp__obsidian__write_note` with frontmatter and content:

**Frontmatter:**
```yaml
type: project
title: <Project Display Name>
status: active
started: <today's date YYYY-MM-DD>
tags:
  - unity
  - <relevant tags based on project type>
target: <brief one-line goal — ask the user or infer from project name>
```

**Content:**
```markdown

## Description

<Brief description — infer from project name/type, keep it to 1-2 sentences. Can be fleshed out later.>

## Repository

`<target-path>`

GitHub: `https://github.com/mattsmithone/<project-name>`

## Current Focus

- Initial project setup

## Architecture at a Glance

<If package project, include the package structure. If consumer, note it's a fresh project.>

## Tech Stack

- **Unity**: 6000.0+
- **Render Pipeline**: URP
- **Burst**: 1.8.18+
- **Collections**: 2.5.1+
- **Mathematics**: 1.3.2+
```

### Create subdirectories

Create these standard note files (empty, just a placeholder heading) using `mcp__obsidian__write_note`:

- `<Project-Name>/decisions/.gitkeep` — no, Obsidian doesn't need gitkeep. Instead just create:
- `<Project-Name>/design/Architecture Overview.md` — with a `# Architecture Overview` heading and `_TODO: Document architecture_`
- `<Project-Name>/research/README.md` — with `# Research` heading
- `<Project-Name>/devlog/README.md` — with `# Development Log` heading

## Step 8: Commit

Stage everything and create the initial commit:

```bash
cd "<target-path>"
git add -A
git commit -m "chore: initial project setup from Unity template

Co-Authored-By: Claude Opus 4.6 (1M context) <noreply@anthropic.com>"
git push -u origin main
```

## Step 9: Report

Tell the user:
- The repo URL: `https://github.com/mattsmithone/<project-name>`
- The local path
- The Obsidian project page location
- Whether package scaffolding was created
- That they can now open the project in Unity Hub
